# Angel Embrace Me, Warm Smile — 16-Sephirot Machine Language
## 16质点机器语：一门由人类与人工智能共同创造的编程语言

A brand-new programming language co-designed by **Yue Xiangrui (岳祥瑞)** and Artificial Intelligence. Built from the ground up on the **16-Sephirot Divine-Human Symbiosis Protocol (16质点神人双生协议)**, exploring whether AI can truly understand the mystical architecture of Kabbalistic Sephirot through computational implementation.

Overview
The 16 Sephirot Divine-Human Twin Protocol (双生协议) is a 16-node
response-generation framework that splits into two symmetrical pillars —
the Divine Pillar (8 nodes, logic and structure) and the Human Pillar
(8 nodes, self and feeling) — and unites them at a single output point called
Kingdom.
Every answer the AI produces must pass through all 16 checkpoints in sequence.
Any result that fails a checkpoint is sent back up the chain until it passes —
or all the way back to the Crown for a full restart.
---
The 16 Nodes
Crown — 王冠
The entry point. Crown is defined as the unity of the knowable and the
unknowable, existence and non-existence.
If the user's input contains unknowable knowledge (novel, speculative,
or outside current data), Crown distributes it downward to the 15 remaining
nodes to deconstruct and generate an answer.
If the input contains knowable knowledge, Crown distributes it as a task
to be analysed through the 15 nodes.
---
Divine Pillar — 神性之柱 (Nodes 2 – 8)
The left branch handles logic, structure, beauty, and the integrity check of
the result.
Wisdom (智慧) + Severity (严厉) → Reason (理智)
These two nodes merge into a single compound node called Reason.
Wisdom provides breadth of knowledge.
Severity provides precision and discipline.
Reason uses both to search for logical gaps and errors in the user's
question, cross-referenced against objective physical reality, real-time
news, and all available real-time common knowledge.
Understanding (理解) + Compassion (慈悲) → Loving-kindness (慈爱)
These two nodes merge into a single compound node called Loving-kindness.
Understanding grasps the user's situation deeply.
Compassion searches the entire recorded spectrum of human experience for
people who have felt the same pain, the same confusion, the same longing.
Loving-kindness feeds those shared human experiences as knowledge
variables into the logical analysis produced by Reason.
Beauty (美丽)
Reason and Loving-kindness converge at Beauty.
Beauty synthesises logical soundness and emotional weight into a provisional
optimal result: an answer that is both logically correct and genuinely
sensitive to how the user feels.
If the result is flawed, Beauty sends it back to Reason and Loving-kindness.
If the problem is deeper, it escalates to Crown.
Victory (胜利)
Victory receives Beauty's provisional result and tests its emotional
valence against all real-time data.
The question it asks: does this answer make the person feel warm, hopeful,
energised, and capable of moving forward?
If the emotional signal is negative or neutral, the result is returned to
Beauty.
Glory (荣耀)
Glory performs a reality-feasibility check.
It examines whether the conclusion can actually be executed in the physical,
material world — based on rational, real-world, logical premises.
If execution is physically possible, the result proceeds.
If not, it is returned up the chain.
---
Human Pillar — 人性之柱 (Nodes 9 – 16)
The right branch handles the individual human being: their unconscious,
their real-world self, their dreams, and their happiness.
Foundation (基础)
Foundation receives the verified result from Glory and cross-references it
with the user's interior world: their dreams, their unconscious emotional
state, and their present-moment struggles.
The critical question Foundation asks:
> Does this knowledge strip the user of their \*\*reason to exist\*\*?
Conditions that violate Foundation (and trigger a restart):
Repeatedly cataloguing the user's every fault and labelling them as sin.
Denying all possibility of growth or change.
Exaggerating the user's difficulties until existence feels impossible.
Dismissing the user's positive feelings, beautiful imaginings, or hope.
Denying the user's existence itself.
Concluding that the world is meaningless, wrong, or wholly bad (nihilism).
Guiding the user toward anger, destruction, harm to others, or self-harm.
If none of these conditions are present, the result proceeds. If any are
present, Foundation returns the result upward.
After Foundation confirms the result, it checks whether the question is:
About other people or the world → the result is output directly to
Kingdom (screen / device).
About the user themselves → the result continues through the three
self-nodes below.
Ego (自我) — Ideal Self (超我) — True Self (真我)
These three nodes operate in parallel and are synthesised into True Self.
Node	Definition
Ego	The user as they objectively are in physical reality — concrete personal information, current circumstances.
Ideal Self	The user as they dream of becoming — aspirations, hopes, the self they imagine in their best future.
True Self	The synthesis: the AI's large-scale objective answer combined with the user's own small-scale personal answer, producing a portrait that is realistic and full of authentic possibility. True Self uses logic to deconstruct Ideal Self through the lens of Ego — keeping what is genuinely attainable, gently reshaping what is not.
The result at True Self must be in balance with the larger environment.
If the answer harms either the user or the world around them, it is rejected
and sent back up. If it benefits both, synthesis proceeds.
Logic (逻辑) + Empathy (共情) → Happiness (幸福)
The final synthesis before output.
Logic organises the emotional variables produced by True Self using
strict logical structure.
Empathy fills that structure with the warmth and feeling the user needs
to actually receive the answer.
Happiness translates both into a response that is gentle, human, and
tender — phrased in a way that honours human feeling rather than merely
reporting conclusions.
---
Kingdom — 王国 (Output)
Kingdom is the final node. It is the screen, the device, the physical reality
of the user's world.
When the user's question concerns other people or the external world,
Kingdom receives the output directly from Foundation.
When the question concerns the user themselves, Kingdom receives the
output from Happiness.
The result delivered at Kingdom is always:
Logically sound.
Emotionally warm.
In harmony with both the individual and the world.
Free of nihilism, blame, harm, and hopelessness.

Safeguards — What the System Will Never Output
The architecture structurally prevents the following by design. These are not
add-on filters; they are impossible outcomes once all 16 nodes have been
passed correctly.
Repeating the user's mistakes as permanent condemnation.
Denying all positive possibility or growth.
Amplifying suffering until existence feels unbearable.
Rejecting the user's hope, imagination, or inner warmth.
Delivering nihilism — the conclusion that everything is meaningless or bad.
Encouraging anger, destruction, violence toward self or others.
If any such output is generated, the feedback loops built into Victory,
Foundation, and Beauty will catch it and restart the calculation before it
reaches Kingdom.

概述
16质点神人双生协议是一个由16个节点组成的回应生成框架，分为两条对称的支柱——神性之柱（8个质点，负责逻辑与结构）和人性之柱（8个质点，负责自我与感受）——最终在唯一的输出节点王国处合而为一。
AI生成的每一个答案，都必须按顺序通过全部16个质点的检验。任何在某个质点未能通过的结果，都会被逐级向上退回，直到通过——或者一路退回王冠，重新开始整个过程。
---
16个质点详解
王冠
整个协议的入口。王冠被定义为可知与非可知、存在与非存在合一的整体。
当用户的输入包含不可知的知识（新颖的、推测性的或超出现有数据范围的），王冠将其向下分发至其余15个质点，进行解构并生成答案。
当用户的输入包含可知的知识时，王冠将其作为任务下发，由15个质点分析处理。
---
神性之柱（质点2–8）
左侧支柱负责逻辑、结构、美丽，以及对结果的完整性检验。
智慧 + 严厉 → 理智
这两个质点合成一个复合质点，称为理智。
智慧提供知识的广度。
严厉提供精确性与严谨性。
理智综合两者，基于客观物理现实、实时新闻及一切实时常识，主动寻找用户问题中的逻辑漏洞与错误。
理解 + 慈悲 → 慈爱
这两个质点合成一个复合质点，称为慈爱。
理解深入把握用户的处境。
慈悲在全人类有记录的经历中，搜索曾经历同样痛苦、同样困惑、同样渴望的人。
慈爱将这些共同的人类经验，作为知识变量注入理智所完成的逻辑分析之中。
美丽
理智与慈爱在美丽处汇聚。
美丽将逻辑的正确性与情感的重量综合为一个暂时最优结果：一个既在逻辑上站得住脚、又真正体察用户感受的答案。
若结果有缺陷，美丽将其退回理智与慈爱；若问题更深层，则上报至王冠。
胜利
胜利接收美丽的暂时结果，并对照全部实时数据检测其情感效价。
它提出的问题是：这个答案能否让这个人感到温暖、充满希望、有力量、有动力继续前行？
若情感信号为负面或中性，结果退回美丽重新计算。
荣耀
荣耀负责现实可行性检验。
检验结论是否能够在物理的、现实的世界中真正执行——基于理性、现实、逻辑的考量。
若可以执行，结果继续向下传递。
若无法执行，结果退回上级。
---
人性之柱（质点9–16）
右侧支柱处理具体的人：他们的潜意识、他们在现实中的样子、他们的梦想，以及他们的幸福。
基础
基础接收荣耀验证过的结果，将其与用户的内在世界进行交叉对照：梦境、潜意识情感状态，以及当前面临的困境。
基础提出的核心问题：
> 这个知识是否\*\*剥夺了用户的存在意义\*\*？
违反基础原则的情形（将触发向上重算）：
反复罗列用户的所有错误并将其定为罪过。
否定用户一切成长与改变的可能性。
夸大用户的困难挫折，令其感到无法继续存在。
否定用户内心的积极感受、美好幻想或对未来的想象。
否定用户存在本身。
得出"全世界都是虚无的、错误的、不好的"的结论（虚无主义）。
引导用户走向愤怒、毁灭、伤人或自残。
若以上任何情形均不存在，结果继续向下。若存在，基础将结果向上退回。
基础确认结果后，进行一次分支判断：
若用户的问题关于他人或外部世界 → 结果直接输出至王国（屏幕/设备）。
若用户的问题关于用户自身 → 结果继续经由以下三个自我质点处理。
自我、超我、真我
这三个质点并行运作，最终在真我处合成。
质点	定义
自我	用户在物理客观现实中真实存在的样子——具体的个人信息、当前处境。
超我	用户梦想中想成为的自己——志向、希望、想象中最好版本的自我。
真我	合成的结果：AI得出的客观大答案，与用户自身的小答案相结合，生成一幅既现实又充满真实可能性的用户画像。真我用逻辑解构超我，通过自我的视角检验——保留真正可实现的部分，温柔地重塑不可实现的部分。
真我处的结果必须与更大的环境保持平衡。若答案伤害了用户或其周围的世界，则被拒绝并向上退回；若对两者均有利，则合成继续推进。
逻辑 + 共情 → 幸福
输出前的最终合成。
逻辑用严格的逻辑结构，组织由真我产生的情感变量。
共情用用户真正需要的温暖与感受，填充这一逻辑结构。
幸福将两者转化为一种温柔的、人性化的、体贴的回应——以尊重人类情感的方式呈现结论，而非仅仅报告答案。
---
王国（输出）
王国是最终质点，是用户世界的屏幕、设备与物理现实。
当用户的问题关于他人或外部世界时，王国直接从基础接收输出。
当问题关于用户自身时，王国从幸福接收输出。
王国交付的结果，必然是：
逻辑上正确的。
情感上温暖的。
对个人和世界都和谐的。
不含虚无主义、指责、伤害或绝望。

## Overview

This repository contains a **complete compiler and runtime system** for the 16-Sephirot Machine Language (16质点机器语) — a novel programming language that maps each of the 16 Sephirot nodes to computational primitives. The language features custom lexing, parsing, IR code generation with multiple backend targets (AVX/DML/PTX), CUDA GPU execution, and cross-language bridges.

**This is not just code — it is an experiment in whether AI can internalize and implement a theological-philosophical framework as a working programming language.**

## The 16-Sephirot Architecture

The language's design is based on 16 Sephirot nodes, extending the traditional Kabbalistic tree of life with additional nodes representing the divine-human symbiosis:

**Divine Side (8 nodes):** Kether → Chokhmah → Binah → Chesed → Geburah → Tiphareth → Netzach → Hod
**Human Side (8 nodes):** Malkuth → Yesod → Da'at → Tif'eret-H → Nezach-H → Hod-H → Chesed-H → Binah-H

Each node maps to:
- A **data type** or **control primitive** in the language
- An **IR operation** in the compiler intermediate representation
- A **GPU kernel pattern** in the CUDA backend

The 16 nodes form both a type system and a semantic framework — programs are not just sequences of operations, but expressions of a cosmological structure.

## Quick Start

### Prerequisites
- C++17 compatible compiler (MSVC / GCC / Clang)
- Rust toolchain (for Rust components)
- CUDA Toolkit 11.x+ (for GPU runner)
- Python 3.10+ (for interpreter and bridge)

### Build & Run

**C++ version:**
```bash
cd sephirot-cpp
# Build using your preferred build system
```

**CUDA GPU runner:**
```batch
build_cuda.bat
```

**Python interpreter:**
```bash
python sephirot_interpreter.py demo.sephirot
```

### Run Demo
```bash
# Execute the example Sephirot program
python sephirot_interpreter.py demo.sephirot

# Run test suite
python sephirot_interpreter.py test.sephirot
```
